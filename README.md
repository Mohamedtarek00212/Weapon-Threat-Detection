# Weapon Threat Detection

A reproducible, notebook-first computer-vision research framework for low-latency weapon threat detection in surveillance imagery. The original dataset is strictly read-only; all derived data and research artifacts are written outside `original/`.

## Dataset

The provided YOLO dataset has seven classes: `Blunt_Weapon`, `Explosive`, `Fire_Smoke`, `Firearm`, `Melee_Weapon`, `Person`, and `Tool`. Its source metadata is preserved in `original/data.yaml`.

## Setup (Python 3.10.11)

```bash
python3.10 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
python -m ipykernel install --user --name weapon-threat-detection --display-name "Weapon Threat Detection"
jupyter lab
```

On Windows, activate with `.venv\\Scripts\\activate`.

## Phase 1 workflow

Run `notebooks/00_Project_Initialization.ipynb` to create a timestamped experiment directory. It records the active configuration, Python and PyTorch versions, selected hardware backend, Git revision when available, and a dedicated log file. It does not inspect, process, or modify the dataset, and it does not train a model.

Future notebooks are present as planned research workflow placeholders and are outside the Phase 1 scope.

## Design guarantees

- `original/` is never modified by project utilities.
- Validation reports corrupted candidates but does not delete or alter any source file.
- Empty label files are treated as valid background samples.
- Device selection automatically prioritizes CUDA, then Apple MPS, then CPU.
- Phase 1 writes timestamped configuration snapshots, metadata, and logs under `runs/`.
- `processed/`, `runs/`, and generated artifacts are excluded from version control.

## Project layout

- `configs/` — versionable settings.
- `src/weapon_threat_detection/` — reusable Python functions only.
- `notebooks/` — notebook-first research workflow.
- `runs/` — timestamped experiment configuration, runtime metadata, and logs.
- `reports/`, `logs/`, `processed/` — generated by later project phases.

## Reproducibility

Set the seed in `configs/project.yaml`. Record the produced JSON/CSV/PNG artifacts with each experiment. For reliable training comparison, retain the generated processed-dataset metadata and the Ultralytics run directory together.
